What is wbia-core?

What it is

Stateless, pure-Python pip package containing the algorithmic primitives for wildlife re-identification. Pure functions over numpy arrays. Zero DB, zero network, zero IBEISController.

What it replaces

wildbook-ia/wbia/algo/hots/ — the 3000-line HotSpotter module inside a 13-package legacy application (IBEISController, depcache, ZMQ job engine, Gunicorn).

What it doesn't do

Detection (YOLO, MegaDetector) and classification — those are ml-service's job. Foreground masking (RF probchip) also belongs in ml-service.

Root Cause Analysis — The 75%→92% Gap

Benchmark bugs (both fixed 06-08)

Bug 1 — wrong score field: normalise_wbia_result read annot_score_list (pre-nsum csum) instead of score_list (post-nsum canonical). Fixed to read score_list.

Bug 2 — missing annot names: run_query() never passed annot_name_list to WBIA's /api/annot/json/, so WBIA auto-generated unique names for all 50 annots. name_groupxs had 50 groups of size 1 — making fmech degenerate to per-annot csum with zero cross-annot aggregation. Fixed: passes COCO name_uuid strings.

Both fixes applied. Needs re-benchmark to verify gap closure.

fmech aggregation gap

wbia-core's fmech gives name 6373 a score of 19.82 by aggregating max-per-qfx matches across all ~7 DB annotations sharing that name. WBIA's nsum gives only 2.97 for the same name — a 6.6× difference.

The csum values match (6189: 11.18 vs 11.08, 1948: 3.09 vs 3.11), confirming FLANN distances are not the problem. The gap is in the fmech aggregation step itself: different per-match data being fed into the MAX-per-qfx-per-name summation.

Why it matters

Name 6373 has ~7× more annotations in the database than the correct name 6189 (1 annot). fmech cherry-picks the best match per query feature across all annotations of a name. With 7× more annots, name 6373 gets an outsized advantage.

WBIA's fmech does NOT show this same amplification, suggesting a difference in how many matches survive into fmech, or how per-match scores are weighted across annots of the same name.

What's Implemented (22 items)

Scoring & filters

  • LNBNN: ndist - vdist
  • bar_l2: 1.0 - vdist
  • ratio: 1.0 - vdist/ndist + threshold
  • FG: sqrt(q_fg × db_fg) (np.ones fallback)
  • const: uniform 1.0 weight
  • normonly: replace vdist with ndist
  • normalizer_rule: 'last' and 'name'

Name-level scoring

  • fmech/nsum_wbia — group by name→qfx, max per group, sum
  • csum_wbia — max per name
  • sumamech — sum per name
  • Canonical alignment — best annot per name gets score
  • rotation_invariance — XY-dedup for rotated features

Spatial verification

  • Prescoring shortlist (top 40 names, 3 annots/name)
  • RANSAC homography via cv2.findHomography
  • xy_thresh=0.01, scale_thresh=2.0, ori_thresh=TAU/4
  • min_inliers=4, use_chip_extent, weight_inliers

Config & pipeline

  • Kpad dynamic / fixed policy
  • can_match_samename toggle (default True)
  • flann_trees=8, flann_checks=800
  • sqrd_dist_on toggle, prescore_method='nsum'
  • minscale/maxscale/fgw_thresh feature filters
  • FLANN global index (matching WBIA NeighborIndex)
